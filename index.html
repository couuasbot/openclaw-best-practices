How it Works

The core of OpenClaw's multi-agent system relies on three pillars:

  • subagents/sessions_spawn: Allows agents to create isolated execution environments for specific tasks. Sub-agents run with scoped toolsets and return results upon completion.
  • Thread Bindings: Connects specific communication threads (Discord, etc.) to a session/sub-agent target, ensuring context continuity.
  • allowAgents: Configuration setting that defines which agent IDs are permitted to be spawned or targeted within a specific context.

Patterns

  • Dispatch v2: A pattern where a "God" agent acts as the central router, delegating execution to runners while specialized agents (COO/CTO/CMO) provide structured consultations.
  • Event-Sourced Workflow: Uses an append-only workflow-events.jsonl as the authoritative state, allowing for recovery from failures and transparent auditing.
  • Cron Healthcheck: Periodic background checks to ensure workflow integrity, detect timeouts, and trigger automated reconciliations.
