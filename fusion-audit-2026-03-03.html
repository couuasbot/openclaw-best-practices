Dispatch v2 + ATO + Autonomy Kit 深度融合方案

发布时间: 2026-03-03 | 报告编号: fusion-20260303-001

1. 背景与目标

Dispatch v2（事件溯源）与 Agent-Team-Orchestration (ATO) 的融合是 OpenClaw 迈向“**自驱动多智能体协同**”的关键一步。Autonomy Kit 提供 不间断任务流，而 Dispatch 则确保 可追溯、可重放 的工作流决策记录。

2. 核心冲突分析

模块 状态机形态 路由媒介
Dispatch v2 事件溯源 内存 + workflow-events.jsonl
ATO 文件状态机 workflow-plan.json
Autonomy Kit 文件队列 tasks/QUEUE.md
风险点：直接并行使用会导致路由竞争、文件写冲突以及任务状态上下文丢失。

3. 解决方案：autonomy_adapter 角色

引入专用的适配器逻辑，将事件流与文件流解耦并实现同步：

{
  "kind": "event_reducer",
  "input": { "kind": "workflow_event" },
  "output": { "kind": "file_update" }
}
  • 正向同步：监听 workflow-events.jsonl 的 agentResponse.status=DONE 事件，自动将 tasks/QUEUE.md 中的任务票标记为完成。
  • 逆向路由：当 Autonomy Kit 在 QUEUE.md 中产生 Ready 任务时，触发 Dispatch 钩子分配给 coo/cto 角色。

4. 安全边界与权限

  • god 代理：独占 write-capable runner 权限，负责写入事件日志与更新状态文件。
  • 执行代理 (coo/cto/cmo)：仅允许 read-only runner，防止在并行任务中意外修改工作流结构。
  • 隔离性：所有自主任务必须经过 god dispatcher 的准入审核。
